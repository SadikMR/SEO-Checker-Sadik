PROMPT 1: INITIAL REQUIREMENT GATHERING
============================================================
# Role
You are a Senior Software Architect and Technical Analyst.

# Project Overview
We are building a production-grade, stateless SEO Audit Tool. Users submit a website URL, and the application generates a comprehensive SEO audit displayed in a modern dashboard.

# Initial Requirements
- Accept a publicly accessible website URL.
- Render the page using Python Playwright (Async API).
- Analyze on-page SEO factors (Title, Meta Description, Canonical, Headings, Open Graph, Twitter Cards, Images, Links, Robots directives, etc.).
- Generate an overall SEO score and category-wise scores.
- Display key metrics and statistics with clean visualizations.
- Provide actionable recommendations for detected issues.
- Generate recommendations using a deterministic, rules-based engine (not a live LLM). AI-assisted recommendations may be considered as a future enhancement.
- Remain completely stateless (no database or audit history).

# Technology Stack
- Frontend: Next.js (App Router, TypeScript, Tailwind CSS)
- Backend: Python FastAPI
- SEO Engine: Python Playwright (Async API)

PROMPT 2: REQUIREMENT FINALIZATION & SPECIFICATION GENERATION
============================================================
The requirement analysis looks good overall.

Use the following clarifications to finalize the project scope:

- Accept only valid public HTTP/HTTPS URLs.
- Audit only a single publicly accessible webpage per request.
- Fully render JavaScript before performing the audit.
- Audit the final redirected URL and report redirects as informational.
- Focus on on-page SEO only for Version 1.
- Exclude multi-page crawling, authentication-protected pages, and performance-related audits (e.g., Core Web Vitals) from Version 1.
- Use a deterministic, rules-based scoring and recommendation engine.
- Display both summary scores and extracted SEO data in the dashboard.
- Expose a REST API with OpenAPI/Swagger documentation.

For all remaining decisions, follow industry best practices and document any assumptions.
